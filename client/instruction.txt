Screen Inventory Template (UX/Product Design)
Screen Identification:
* Screen ID - Unique reference (e.g., S-001, HOME-01)
* Screen Name - Descriptive title (e.g., "Login Screen", "Product Listing Page")
* Screen Type - Landing, Form, List, Detail, Modal, Error, Success, etc.
User Journey Context:


* User Role/Persona - Who uses this screenFunctional Elements:
* Primary Action - Main CTA or goal (e.g., "Submit order", "Create account")
* Secondary Actions - Other available actions (Cancel, Back, Skip)
* Input Fields - What data users provide
* Content Displayed - Key information shown to users
* Validation/Rules - Business logic, field requirements


Additional Information:
* Dependencies - Other screens or features needed first
* Edge Cases - Error states, empty states, loading states
